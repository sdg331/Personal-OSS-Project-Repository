# Git 원격 저장소 복제 (Clone)

## 학습 목표

  - GitHub에서 원격 저장소를 생성할 수 있다
  - 원격 저장소를 지역 저장소에 복제할 수 있다
  - 원격 저장소의 별칭 이름을 이해하고 관리할 수 있다
  - VSCode에서 원격 저장소를 복제할 수 있다

-----

## 1\. Clone 개념 및 원격 저장소

### 원격 저장소란

  - **GitHub**: `https://github.com/git`
  - **Bitbucket**: 또 다른 원격 저장소 서비스
  - 클라우드 기반 중앙 저장소(Central Repository/upstream)

### Clone의 개념

  - 원격 저장소를 지역 저장소로 복제하는 작업
  - 공개된 저장소는 소유와 관계없이 누구나 복제 가능
  - GitHub(Cloud) → Local Computer로 전체 프로젝트 복사

-----

## 2\. GitHub 원격 저장소 생성

### 저장소 생성 단계

  - GitHub 계정으로 로그인
  - 새 저장소 생성 (Repository name: `git-clone`)
  - **Add a README file** 체크
  - **Public/Private** 선택
  - `.gitignore` 및 License 설정 (선택사항)

### 저장소 주소 복사

  - GitHub 저장소 페이지에서 **Code** 버튼 클릭
  - HTTPS 주소 복사 (예: `https://github.com/ai7dnn/git-clone.git`)

-----

## 3\. Git Clone 명령어

### 기본 복제 명령어

#### 원격 저장소와 동일한 이름으로 복제

```bash
$ git clone https://github.com/ai7dnn/git-clone.git
```

  - 현재 디렉토리 하위에 `git-clone` 폴더 생성 후 복제
  - 실행 결과:
    ```
    Cloning into 'git-clone'...
    remote: Enumerating objects: 3, done.
    remote: Counting objects: 100% (3/3), done.
    remote: Total 3 (delta 0), reused 0 (delta 0), pack-reused 0
    Receiving objects: 100% (3/3), done.
    ```

#### 다른 폴더명으로 복제

```bash
$ git clone https://github.com/ai7dnn/git-clone.git git-clone-test
```

  - `git-clone-test`라는 폴더명으로 복제
  - 원격 저장소 이름과 다른 로컬 폴더명 사용 가능

#### 현재 폴더에 바로 복제

```bash
$ git clone https://github.com/ai7dnn/git-clone.git .
```

  - 하위 폴더 생성 없이 현재 디렉토리에 직접 복제
  - 마지막에 `.` (점) 사용

## 4\. 원격 저장소 별칭 관리

### 원격 저장소 확인

#### 원격 저장소 목록

```bash
$ git remote
```

  - 출력: `origin`
  - 기본 별칭 이름 `origin` 자동 설정

#### 원격 저장소 상세 정보

```bash
$ git remote -v
```

  - 출력 예시:
    ```
    origin https://github.com/ai7dnn/git-clone.git (fetch)
    origin https://github.com/ai7dnn/git-clone.git (push)
    ```
  - 저장소 주소 및 fetch/push 정보 표시

### 원격 저장소 관리 명령어

#### 원격 저장소 추가

```bash
$ git remote add origin URL
```

  - 새로운 원격 저장소 별칭 저장

#### 상세 정보 조회

```bash
$ git remote show origin
```

  - 출력 예시:
    ```
    * remote origin
      Fetch URL: https://github.com/ai7dnn/oss-lect.git
      Push  URL: https://github.com/ai7dnn/oss-lect.git
      HEAD branch: main
      Remote branch:
        main tracked
      Local branch configured for 'git pull':
        main merges with remote main
      Local ref configured for 'git push':
        main pushes to main (up to date)
    ```
  - 모든 브랜치, URL, HEAD branch 정보 확인 가능

#### 원격 저장소 이름 변경

```bash
$ git remote rename origin org
```

  - `origin`을 `org`로 변경
  - 확인: `Renaming remote references: 100% (3/3), done.`

#### 원격 저장소 삭제

```bash
$ git remote rm org
```

  - 별칭 `org` 삭제

### 복수 원격 저장소 관리 예시

```bash
$ git remote -v
origin https://github.com/ai7dnn/oss-lect.git (fetch)
origin https://github.com/ai7dnn/oss-lect.git (push)
vscode https://github.com/torvalds/linux (fetch)
vscode https://github.com/torvalds/linux (push)
```

```bash
$ git remote rename vscode mcode
```

```bash
$ git remote -v
mcode https://github.com/torvalds/linux (fetch)
mcode https://github.com/torvalds/linux (push)
origin https://github.com/ai7dnn/oss-lect.git (fetch)
origin https://github.com/ai7dnn/oss-lect.git (push)
```

## 6\. VSCode로 원격 저장소 복제

### GUI 복제 절차

#### 1단계: 폴더 준비

  - 적당한 폴더 생성 (예: `C:\[OSS]`)
  - 우클릭 → **Code(으)로 열기** 선택

#### 2단계: Clone Git Repository 선택

  - VSCode 시작 화면에서 **Clone Git Repository** 클릭
  - 또는 Command Palette에서 "Git: Clone" 검색

#### 3단계: 저장소 URL 입력

  - 팔레트에 Git 주소 입력
    ```
    https://github.com/git/git.git
    ```
  - **Clone from URL** 또는 **Clone from GitHub** 선택 가능

#### 4단계: 저장 위치 지정

  - 복제할 폴더 선택 (예: `C:\[OSS]`)
  - **Select as Repository Destination** 클릭
  - 하위에 저장소 이름으로 폴더 자동 생성

#### 5단계: 복제 진행

  - 진행 상태 표시: `Cloning git repository "https://github.com/git/git.git"...`
  - 완료 후 선택:
      - **Open**: 현재 VSCode 창에서 열기
      - **Open in New Window**: 새 창에서 열기
      - **Add to Workspace**: 현재 작업 공간에 추가

### VSCode Terminal에서 복제

```bash
$ git clone https://github.com/microsoft/vscode.git
```

  - VSCode 내장 Terminal(Bash) 사용 가능
  - CLI와 동일한 명령어 실행

## 핵심 명령어 요약

| 명령어 | 기능 |
|--------|------|
| `git clone [URL]` | 원격 저장소 복제 (동일 이름) |
| `git clone [URL] [폴더명]` | 지정한 폴더명으로 복제 |
| `git clone [URL] .` | 현재 폴더에 직접 복제 |
| `git remote` | 원격 저장소 목록 조회 |
| `git remote -v` | 원격 저장소 상세 정보 조회 |
| `git remote show origin` | 특정 원격 저장소 자세한 정보 |
| `git remote add origin [URL]` | 원격 저장소 추가 |
| `git remote rename [이전] [새이름]` | 원격 저장소 이름 변경 |
| `git remote rm [이름]` | 원격 저장소 삭제 |

-----

## 주요 학습 포인트

  - Clone은 원격 저장소 전체를 로컬로 복사하는 작업이다
  - 기본 원격 저장소 별칭은 `origin`으로 자동 설정된다
  - 복제 시 폴더명을 지정하지 않으면 저장소 이름으로 자동 생성된다
  - VSCode에서 GUI로 복제하면 더 직관적이고 편리하다
  - `git remote` 명령어로 여러 원격 저장소를 관리할 수 있다
  - 공개 저장소는 누구나 복제 가능하나, Push 권한은 별도로 필요하다
