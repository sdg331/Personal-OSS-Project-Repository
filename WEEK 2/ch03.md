## 깃(Git) 개요

### Git의 탄생과 역사

#### 개발자
- **2005년**: 리누스 토발즈(Linus Torvalds)가 개발
- **현재 유지보수**: 주니오 하마노(Junio Hamano)

#### 대표적인 오픈소스 프로젝트
- GitHub 저장소: https://github.com/git
- 공식 저장소: https://git.kernel.org/pub/scm/git/git.git

### Git의 정의

#### 핵심 정의
- 컴퓨터 파일의 변경을 추적하는 데 사용되는 **분산 버전 관리 시스템**
- A version control system to keep track of changes to files and projects over time

### Git 사용 장점

#### 분산 버전 관리의 강점
- 모든 개발자는 **지역 시스템에 코드의 전체 사본을 소유**
- 소스 코드에 대한 모든 변경 사항은 **다른 사용자가 추적 가능**

#### 협업 구조
```
Business Organization
    ├── Developer 1 (Java)
    ├── Developer 2 (MySQL)
    └── Developer 3
           ↓
    Git Distributed Server
```

### Git의 주요 기능

#### 협업 기능
- 여러 개발자가 함께 작업 가능
- 소스 코드의 변경 사항을 추적하는 데 사용

#### 기술적 특징
- 소스 코드 관리에 **분산 버전 제어 도구** 사용
- **여러 개의 평행 분기**를 통해 비선형 개발 지원

## Git 설치 및 인터페이스

### Git 설치 시 제공되는 두 가지 소프트웨어

#### 1. Git Bash (CLI: Command Line Interface)

##### 특징
- 명령 행 인터페이스
- 텍스트 기반 명령어 입력 방식
- **처음엔 어렵지만, CLI를 사용할 줄 알면 GUI도 사용 가능**
- 반대는 성립하지 않음 (중요!)

##### 운영체제별 대응
- **Mac**: Terminal
- **Windows**: CMD 또는 Powershell

##### 예시
```bash
zicza@DESKTOP-7RDBL75 MINGW64
$ pwd
/c/Users/zicza
```

#### 2. Git GUI (Graphical User Interface)

##### 특징
- 그래픽 대화 화면에서 마우스와 텍스트 입력 방식
- GUI 프로그램의 대부분은 **Git 기능 중 일부만 구현**하기 때문에 비교적 단순

##### 주요 기능
- Create New Repository
- Clone Existing Repository
- Open Existing Repository
- Open Recent Repository

## Git의 내부 저장소 구조

### 깃 내부 주요 영역

#### 1. 작업 디렉토리 (Working Directory)
- **별칭**: Working Folder, Work Space, Working Tree
- **상태**: modified, untracked
- 실제 파일을 수정하는 작업 공간

#### 2. 스테이징 영역 (Staging Area)
- **별칭**: Stage Area, Index
- **상태**: staged, indexed
- 커밋할 파일을 준비하는 중간 단계

#### 3. Git 저장소 (Git Repository)
- **별칭**: Repository, .git Directory
- **상태**: Committed
- 버전 이력이 저장되는 최종 저장소

#### 4. 임시 저장소 (Stash)
- **상태**: stash
- 작업 중인 변경사항을 임시로 저장

### 깃에서 상태 간 이동 명령어

#### 명령어 흐름도
```
Work Space → add → Index(Stage) → commit → Local Repository
                                              ↓ push
                                          Remote Repository
                                              ↓ pull
                                          Local Repository
```

#### 주요 명령어
- **add**: Working Directory → Staging Area로 이동
- **commit**: Staging Area → Git Repository로 이동
- **push**: Local Repository → Remote Repository로 업로드
- **pull**: Remote Repository → Local Repository로 다운로드

## 브랜치 (Branch) 개념

### 브랜치의 의미

#### 사전적 의미
- 나무 가지
- 지점
- 분기

### 브랜치의 개념

#### 실생활과의 비교
- 여러 파일을 제목의 변형으로 버전 관리
- 파일 집합을 다시 버전 시작
- **새로운 폴더에 복사해서 사용하는 것과 비슷**

#### 기술적 정의
- **새로운 수정을 할 수 있는 또 다른 버전의 작업 흐름(workflow)**
- 버전 관리를 수행하던 일련의 파일 집합을 **통째로 복사**해 독립적으로 다시 개발을 진행하는 개념

### 비선형 개발 작업 흐름

#### 브랜치와 병합
```
VERSION CONTROL
    main ──────●──────●──────●──────●
               │              │
               └── branch ────┘
                   (병합 merge)
```

#### 특징
- 여러 개의 평행 분기를 통해 **비선형 개발** 지원
- 각 브랜치는 독립적으로 개발 가능
- 병합(merge)을 통해 통합

## 깃허브 (GitHub) 이해

### GitHub의 정의

#### 핵심 개념
- **Git을 위한 웹 저장소(Repository)**
- A website that hosts Git repositories online, making it easier for developers to share code

#### 역할
- 버전 관리를 위한 서버 저장소
- 프로젝트 개발을 위한 **협업 관리 서비스**
- 커뮤니티 협의 공간
- 시스템 개발자와 운영 담당자 사이의 소통, 협업, 통합 및 자동화 지원

### Git vs GitHub

| 구분 | Git | GitHub |
|------|-----|--------|
| 정의 | 분산 버전 제어 소프트웨어 | Git을 위한 웹 저장소 |
| 역할 | 버전 관리 도구 | 협업 플랫폼 |
| 위치 | 로컬 시스템 | 클라우드 서버 |
| 기능 | 파일 변경 추적 | 코드 공유 및 협업 |

### GitHub 서비스

#### 제공 서비스
- 분산 환경 소스코드 버전 관리 도구인 **Git의 글로벌 웹 호스팅 서비스**
- **세계 최대의 오픈소스 공유 플랫폼**
- 공개/비공개 저장소, 코드 리뷰, 문서화, 커뮤니티 등 제공

#### 저장소 유형
- **무료 서비스**: 공개 저장소만 사용 가능
  - 모두에게 소스 코드 내용 오픈
- **유료 서비스**: 비공개 저장소 사용 가능

### GitHub의 규모

#### 통계 (2018년 8월 기준)
- **9,600만 개 이상**의 프로젝트 호스팅
- **3,100만 명 이상**의 개발자 참여

#### 마이크로소프트 인수
- **2018년 6월 4일**: 마이크로소프트가 **75억 달러(약 9조원)**에 인수 발표
- 현재 마이크로소프트 자회사로 운영

### Git + GitHub = Open Source Software

#### 오픈소스 생태계
- Git과 GitHub를 통해 오픈소스 소프트웨어 개발
- **Social Coding** 플랫폼

#### 대표적인 오픈소스 프로젝트
- PHP
- CakePHP
- openSUSE
- MongoDB
- MySQL
- OpenTok 등

## 핵심 요약 (Summary)

### Git
- 시간에 따른 파일 및 프로젝트의 변경 사항을 추적하는 버전 관리 시스템
- A version control system to keep track of changes to files and projects over time

### GitHub
- 온라인에서 Git 저장소를 호스팅하는 웹사이트
- 개발자가 코드를 쉽게 공유할 수 있도록 지원
- A website that hosts Git repositories online, making it easier for developers to share code

### Repositories (Repo)
- 커밋이라는 의도적인 스냅샷을 포함하는 폴더
- Folders which contain intentional snapshots of progress called commits

### 주요 명령어 정리
- **add**: Working Directory → Staging Area
- **commit**: Staging Area → Git Repository
- **push**: Local → Remote
- **pull**: Remote → Local

### 브랜치
- 버전 관리를 수행하던 일련의 파일 집합을 통째로 복사해 독립적으로 다시 개발을 진행하는 개념

### GitHub의 역할
- 버전 관리를 위한 서버 저장소 및 프로젝트 개발을 위한 협업 관리 서비스
