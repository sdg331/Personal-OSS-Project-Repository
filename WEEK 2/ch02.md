## 버전관리 이해

### 버전 관리의 필요성

#### 기존 관리 방식의 문제점
- 파일 이름으로 버전 관리 시도
  - `120525_문서_업데이트.txt`
  - `120605_문서_수정판.txt`
  - `120605_문서_수정판2.txt`
  - `120605_문서_최신.txt`
- 파일이 많아지면 관리가 혼란스러움
- 무엇이 최신 버전인지 파악 어려움
- 여러 폴더에 파일을 복사하여 관리하는 비효율적 방식

### 버전 관리의 개념

#### 파일 변경 이력 추적 관리
- **누가** (다중 사용자)
- **어느 저장소에서** (저장소가 여러 개인 경우)
- **어느 파일을**
- **언제**
- **어디를**
- **어떻게** (추가/수정/내용)

#### 버전 관리 시스템의 정의
- 시간 흐름에 따라 파일 집합에 대한 변경 사항을 추적, 관리
- The version control system tracks changes to a file or set of files over time

### 버전 관리의 주요 기능

#### 버전 저장과 백업
- 변경사항의 자세한 확인
- 버전 이력 저장

#### 다중 사용자 관리
- 여러 사용자에 대한 버전 이력 추적 관리
- 소스 내용의 충돌 처리
- 어떤 파일이 언제 어떻게 삭제되고 추가됐는지 확인

#### 시간 여행 기능
- 필요하면 과거 어느 시점으로 이동 가능
- 마치 "시간 여행"과 같은 기능

## 버전관리 도구의 인터페이스

### 명령어 줄 인터페이스(CLI: Command Line Interface)

#### 정의
- 텍스트(문자)로 명령을 입력하고 결과도 텍스트로 표시되는 방식
- 윈도의 명령 프롬프트(Command Prompt)
- 유닉스의 쉘(Shell)

#### 예시 명령어
```bash
$ git init repo
$ cd repo
$ git config --global core.autocrlf true
$ git config --global core.safecrlf false
$ git config --get core.safecrlf
$ git config --get core.autocrlf
```

### 그래픽 사용자 인터페이스(GUI: Graphic User Interface)

#### 정의
- 그래픽 대화 화면에서 마우스와 텍스트 입력으로 명령 수행
- 윈도와 같은 방식의 인터페이스

#### 대표적 도구: 소스트리(SourceTree)
- 커밋, Pull, Push, 브랜치, 병합 등을 시각적으로 관리
- 그래프로 브랜치 구조 확인 가능
- 파일 상태를 시각적으로 표시

## 커밋과 버전관리

### 커밋(Commit)의 개념

#### 사전적 의미
- "적어두다"라는 뜻

#### 버전관리에서의 커밋
- 저장소의 현 상태를 저장하는 행위
- 현 상태를 담는 스냅샷 사진을 찍는 것에 비유
- 파일 집합의 변경 내용을 깃 저장소에 기록하는 작업
- 어느 시점의 파일 집합(폴더)의 추가/변경 사항을 저장소에 기록
- 이전 커밋 상태부터 현재 상태까지의 변경 이력이 기록된 커밋이 생성

### 커밋의 시간순 저장

#### 커밋 타임라인
- 시간순으로 저장됨
- 최근 커밋부터 거슬러 올라가면 과거 변경 이력과 내용 확인 가능
- 예시:
  - 2013/08/01 12:06 - 커밋 1
  - 2013/08/01 17:31 - 커밋 2
  - 2013/08/02 14:13 - 커밋 3
  - 2013/08/04 11:52 - 커밋 4

### 커밋 추적 관리

#### 효율적인 저장 방식
- 저장소의 현재 상태를 저장
- 연속된 커밋으로 관리
- 파일이 달라지지 않았으면 파일을 새로 저장하지 않음
- 단지 이전 상태의 파일에 대한 링크만 저장

### HEAD 포인터

#### HEAD의 역할
- 가장 최근의 커밋을 가리키는 포인터
- 현재 작업 중인 위치를 나타냄
- 버전별로 파일 변경 이력을 추적

## 저장소의 개념

### 저장소(Git Repository)

#### 정의
- 파일이나 폴더를 저장해 두는 곳
- 파일이 변경 이력별로 구분되어 저장됨
- 버전관리를 위한 관련 파일을 저장

### 원격저장소와 지역저장소

#### 원격 저장소(Remote Repository)
- 파일이 원격 저장소 전용 서버에서 관리
- 여러 사람이 함께 공유하기 위한 저장소
- 협업 작업의 중심 저장소

#### 지역 저장소(Local Repository)
- 내 PC에 파일이 저장되는 개인 전용 저장소
- 개인 작업 공간
- 원격 저장소의 복제본

## 주요 Git 명령어

### Clone (복제)
- 서버의 원격 저장소를 지역 저장소에 복제
- 처음 프로젝트를 시작할 때 사용
- 한 PC에 다른 저장소로 연습 가능

### Push (업로드)
- 지역 저장소에서 원격 저장소로 올리기
- 평소에는 내 PC의 지역 저장소에서 작업
- 작업한 내용을 공개하고 싶을 때 원격 저장소에 업로드

### Pull (다운로드)
- 원격 저장소에서 지역 저장소로 내리기
- 원격 저장소에서 여러 사람이 작업한 파일을 가져옴
- 최신 변경사항을 동기화

### Add (스테이징)
- 파일을 저장소에 저장하기 위한 준비 단계
- 커밋할 파일을 선택

### Commit (확정)
- 파일을 저장소에 저장하는 명령어
- 변경사항을 확정하여 기록
- 서버의 원격 저장소에서도 add, commit 명령 수행

## 명령어 흐름도

### 기본 워크플로우
```
원격 저장소(Remote Repository)
         ↓ clone
지역 저장소(Local Repository)
         ↓ add
스테이징 영역(Staging Area)
         ↓ commit
지역 저장소 확정
         ↓ push
원격 저장소 업데이트
         ↓ pull
최신 변경사항 동기화
```
