# 27. Visual Studio Code에서 Git 활용 (VS Code Git Integration)

## 1. VS Code 기초 및 환경 설정
### 1.1 개요
* **Visual Studio Code (VS Code)**: 마이크로소프트에서 개발한 가볍고 강력한 소스 코드 편집기. Git 기능이 기본적으로 내장되어 있음.
* **목표**: 터미널 명령어를 일일이 치지 않고도 GUI를 통해 직관적으로 Git을 다루는 법을 익힘.

### 1.2 저장소 초기화 (Init)
* **폴더 열기**: `File` > `Open Folder` (단축키: `Ctrl + K, Ctrl + O`)
* **터미널 열기**: `Terminal` > `New Terminal` (단축키: `` Ctrl + ` ``)
* **Git 초기화**:
    1.  **GUI 방식**: 왼쪽 '소스 제어(Source Control)' 탭 > `Initialize Repository` 버튼 클릭.
    2.  **CLI 방식**: 터미널에 `$ git init` 입력.

---

## 2. 파일 상태 확인 및 커밋 (Basic Workflow)
### 2.1 소스 제어 탭 (Source Control)
* **진입**: 좌측 사이드바의 갈라진 가지 모양 아이콘 (단축키: `Ctrl + Shift + G`).
* **파일 상태 표시 (Decoration)**:
    * **U (Untracked)**: 새로 생성되었으나 아직 Git이 추적하지 않는 파일.
    * **M (Modified)**: 수정된 파일.
    * **A (Added)**: 스테이징 영역에 추가된 파일.

### 2.2 스테이징 (Staging / `git add`)
* **CLI 명령어**: `$ git add [파일명]`
* **VS Code 동작**:
    * '변경 사항(Changes)' 목록에 있는 파일 옆의 **`+` (Plus)** 버튼 클릭.
    * 클릭 시 해당 파일이 **'스테이징된 변경 사항(Staged Changes)'** 영역으로 이동함.

### 2.3 커밋 (Commit / `git commit`)
* **CLI 명령어**: `$ git commit -m "메시지"`
* **VS Code 동작**:
    1.  소스 제어 탭 상단 입력창에 커밋 메시지 작성.
    2.  **`Commit` (커밋)** 버튼(V 체크 모양) 클릭.
    * *Tip*: `Ctrl + Enter`를 누르면 버튼 클릭 없이 바로 커밋됨.

---

## 3. Git Graph 확장 프로그램 활용 (Log Visualization)
### 3.1 Git Graph 설치 및 실행
* **설치**: 확장(Extensions) 탭 > "Git Graph" 검색 > Install.
* **실행**: 하단 상태 표시줄의 `Git Graph` 문구 클릭.

### 3.2 주요 기능 (`git log` 대체)
* **시각화**: 터미널의 `$ git log --graph --oneline --all` 명령어를 예쁜 GUI 그래프로 보여줌.
* **정보 확인**:
    * 커밋 해시(ID), 작성자, 날짜, 메시지 확인 가능.
    * 브랜치(Branch)의 흐름과 병합(Merge) 상태를 직관적으로 파악 가능.
    * 각 커밋을 클릭하면 해당 시점의 파일 변경 내용을 볼 수 있음.

## 4. 파일 비교 기능 (Diff Tool) 상세 정리
*VS Code의 강력한 기능 중 하나는 `git diff` 명령어를 시각적으로 보여주는 것입니다.*

### 4.1 작업 트리 vs 스테이징 영역 비교 (`git diff`)
* **상황**: 파일을 수정했지만 아직 `add` 하지 않은 상태.
* **CLI**: `$ git diff`
* **VS Code**:
    * 소스 제어 탭 > **'변경 사항(Changes)'** 목록.
    * 파일을 클릭하면 **[좌측: 원본] vs [우측: 수정본]** 비교 화면이 열림.
    * 삭제된 줄은 빨간색, 추가된 줄은 초록색으로 표시.

### 4.2 스테이징 영역 vs 최신 커밋 비교 (`git diff --staged`)
* **상황**: 파일을 `add` 했지만 아직 `commit` 하지 않은 상태.
* **CLI**: `$ git diff --staged` (또는 `--cached`)
* **VS Code**:
    * 소스 제어 탭 > **'스테이징된 변경 사항(Staged Changes)'** 목록.
    * 파일을 클릭하면 스테이징된 내용과 이전 커밋 내용을 비교.

### 4.3 작업 트리 vs 최신 커밋 비교 (`git diff HEAD`)
* **상황**: `add` 여부와 상관없이 현재 작업 중인 내용과 마지막 커밋의 차이를 보고 싶을 때.
* **CLI**: `$ git diff HEAD`
* **VS Code (Git Graph 활용)**:
    * Git Graph 화면 최상단의 **`Uncommitted Changes`** 클릭.
    * 작업 중인 모든 변경 사항을 한눈에 비교 가능.

### 4.4 변경 사항 탐색 꿀팁 (Navigation)
* **단축키 `F7`**: 파일 비교 창에서 `F7`을 누르면 다음 변경 사항(Diff)으로 점프.
* **단축키 `Shift + F7`**: 이전 변경 사항으로 점프.
* **인라인 보기(Inline View)**: 우측 상단 메뉴(...)에서 `Inline View`를 켜면 좌우 분할 대신 한 화면에 변경 사항을 표시해 줌.

---

## 5. 핵심 요약 (Summary)
1.  **생산성 향상**: 터미널 명령어 대신 VS Code의 **소스 제어 탭(`Ctrl+Shift+G`)**을 활용하면 `add`, `commit`을 빠르게 수행할 수 있다.
2.  **시각화**: **Git Graph** 확장을 통해 복잡한 커밋 히스토리를 쉽게 파악한다.
3.  **파일 비교**:
    * **Changes 클릭** = `$ git diff`
    * **Staged Changes 클릭** = `$ git diff --staged`
    * **F7 키**를 사용하여 변경된 코드 라인만 빠르게 훑어볼 수 있다.
