#### Git 소개
- **무료 오픈소스 분산 버전 관리 시스템**

- 소규모부터 대규모 프로젝트까지 빠르고 효율적으로 처리하도록 설계

### Git 홈페이지 둘러보기

#### About 섹션 (https://git-scm.com/about)

##### Git의 주요 특징
- **Branching and Merging** (브랜치와 병합)
- **Small and Fast** (작고 빠름)
- **Distributed** (분산형)
- **Data Assurance** (데이터 보장)
- **Staging Area** (스테이징 영역)
- **Free and Open Source** (무료 오픈소스)

##### 브랜치 모델의 장점
Git을 다른 SCM과 차별화하는 핵심 기능은 브랜치 모델이다.[1]

**브랜치 활용 사례**:
- **Frictionless Context Switching** (마찰 없는 컨텍스트 전환)
  - 아이디어 테스트를 위한 브랜치 생성
  - 몇 번의 커밋 후 원래 위치로 돌아가기
  - 패치 적용 후 실험 브랜치로 다시 전환
  - 완료 후 병합

- **Role-Based Codelines** (역할 기반 코드라인)
  - 프로덕션 전용 브랜치
  - 테스트용 병합 브랜치
  - 일상 작업용 여러 개의 작은 브랜치

- **Feature Based Workflow** (기능 기반 워크플로우)
  - 새로운 기능마다 브랜치 생성
  - 기능 간 원활한 전환
  - 완료 후 메인 라인에 병합하고 브랜치 삭제

- **Disposable Experimentation** (일회용 실험)
  - 실험용 브랜치 생성
  - 작동하지 않으면 삭제
  - 다른 사람에게 보이지 않음

## 깃 설치

### 설치 파일 실행

#### 1단계: 라이선스 동의
```
Git 2.38.1 Setup
Information
GNU General Public License Version 2, June 1991
```
- GNU General Public License 표시
- **Next** 클릭하여 계속

#### 2단계: 설치 위치 선택
```
Select Destination Location
C:\Program Files\Git
```
- **기본 경로**: `C:\Program Files\Git`
- **필요 공간**: 최소 294.6 MB
- 다른 폴더 선택 시 **Browse** 클릭
- **Next** 클릭하여 계속

#### 3단계: 컴포넌트 선택
```
Select Components
```
**추천 옵션**:
- ☑ Additional icons - On the Desktop
- ☑ Windows Explorer integration
  - ☑ Git Bash Here
  - ☑ Git GUI Here
- ☑ Git LFS (Large File Support)
- ☑ Associate .git* configuration files with the default text editor
- ☑ Associate .sh files to be run with Bash
- ☑ Check daily for Git for Windows updates
- ☑ (NEW!) Add a Git Bash Profile to Windows Terminal

### 편집기 설정

#### 기본 편집기 선택
```
Choosing the default editor used by Git
```

**편집기 옵션**:
- Use Vim (the ubiquitous text editor)
- Use the Nano editor
- Use Notepad++ as Git's default editor
- **Use Visual Studio Code as Git's default editor** ✅ **추천**
- Use Visual Studio Code Insiders
- Use Sublime Text
- Use Atom
- Select other editor

**권장 설정**: Visual Studio Code를 기본 편집기로 선택[1]

### 초기 브랜치 이름 설정

#### 새 저장소의 기본 브랜치 이름
```
Adjusting the name of the initial branch in new repositories
```

**옵션 1**: Let Git decide
- Git의 기본 브랜치 이름 사용 (현재: "master")
- Git 프로젝트는 향후 더 포용적인 이름으로 변경 예정

**옵션 2**: Override the default branch name ✅ **추천**
- 새 저장소의 기본 브랜치 이름 재정의
- 많은 팀이 이미 기본 브랜치 이름을 변경함
- **일반적인 선택**: "main", "trunk", "development"
- **권장 설정**: `main` 입력[1]

**명령어로 설정**:
```bash
$ git config --global init.defaultBranch main
```

⚠️ **주의**: 이 설정은 기존 저장소에는 영향을 주지 않음

### 설치 완료

#### 마지막 단계
```
Completing the Git Setup Wizard
```

**옵션**:
- ☑ Launch Git Bash (Git Bash 실행)
- ☐ View Release Notes (릴리즈 노트 보기)

**Finish** 클릭하여 설치 완료

## 3️⃣ 깃 실행

### 실행 방법 1: 시작 메뉴

#### Windows 시작 메뉴에서 실행
```
검색: git bash
```

**표시되는 항목**:
- **Git Bash** ✅ (주로 사용)
- Git CMD
- Git GUI
- Git FAQs (Frequently Asked Questions)
- Git Release Notes

**실행**: "Git Bash" 앱 클릭

### 실행 방법 2: 탐색기에서 우클릭

#### 탐색기 폴더에서 실행
1. 원하는 폴더로 이동
2. 폴더 내 빈 공간에서 **우클릭**
3. 컨텍스트 메뉴에서 **"Git Bash Here"** 선택

**장점**: 원하는 폴더에서 바로 Git Bash를 실행 가능

**예시**:
```
탐색기 경로: C:\OSS\git
우클릭 메뉴:
  - 터미널에서 열기(T)
  - Open with GitKraken
  - Git GUI Here
  - Git Bash Here ✅
  - Code(으)로 열기
```

**실행 결과**:
```bash
PC@DESKTOP-482NOAB MINGW64 /c/oss/git
$ pwd
/c/oss/git
```

### 실행 방법 3: 경로창 직접 입력

#### 탐색기 주소창에서 실행
1. 탐색기에서 원하는 폴더로 이동
2. 폴더 경로가 표시되는 주소창 클릭
3. 직접 **`git bash`** 입력 후 Enter

**장점**: 가장 빠른 실행 방법

## 4️⃣ CLI 인터페이스 이해

### Command Line Interface (CLI)

#### CLI의 특징
- 명령 프롬프트와 같은 인터페이스
- 텍스트 기반 명령어 입력 방식
- 리눅스 명령어 사용 가능

#### 기본 리눅스 명령어

##### 현재 디렉토리 확인
```bash
$ pwd
/c/[git tutorial]
```
- `pwd`: Print Working Directory
- 현재 작업 중인 디렉토리 경로 출력

##### 파일 목록 확인
```bash
$ ls -al
total 8
drwxr-xr-x 1 PC 197121 0 May 22 10:12 ./
drwxr-xr-x 1 PC 197121 0 May 22 10:13 ../
```
- `ls`: List (목록 보기)
- `-a`: All (숨김 파일 포함)
- `-l`: Long format (자세한 정보)

### Git 명령어 기초

#### 프롬프트 구조
```bash
PC@DESKTOP-482NOAB MINGW64 /c/[git tutorial]
└─┬─┘ └───────┬──────┘ └─┬──┘ └──────┬──────┘
사용자    컴퓨터 이름    터미널   현재 디렉토리
```

#### 버전 확인
```bash
$ git --version
git version 2.35.1.windows.2
```
- Git 설치 버전 확인
- 설치 성공 여부 검증

#### 설정 확인
```bash
$ git config --list
```
- Git 전역 및 로컬 설정 목록 표시
- 사용자 정보, 기본 브랜치 등 확인

#### 도움말 보기
```bash
$ git --help
```
- Git 명령어 도움말 표시
- 주요 명령어 목록 및 사용법 안내

## 핵심 요약 (Summary)

### 깃 설치 다운로드
- **공식 사이트**: https://git-scm.com/downloads
- **파일명**: Git-2.39.0.2-64-bit.exe (버전에 따라 다름)

### 깃 설치 과정
1. 설치 파일 실행
2. 라이선스 동의
3. 설치 위치 선택 (기본: `C:\Program Files\Git`)
4. 컴포넌트 선택
5. **편집기 설정**: Visual Studio Code 추천
6. **초기 브랜치 이름**: `main`으로 설정 추천
7. 계속 **Next** 클릭하여 설치 진행
8. **Finish**로 완료

### 설치 성공 확인
- **설치 경로**: `C:\Program Files\Git`
- **폴더 구조**: bin, cmd, mingw64, usr 등

### 깃 실행 방법
1. **시작 메뉴**: Git Bash 검색 후 실행
2. **탐색기 우클릭**: "Git Bash Here" 선택
3. **주소창 직접 입력**: 폴더 경로창에 `git bash` 입력

### CLI 이해 및 기본 명령어
```bash
$ git --version    # 버전 확인
$ git version      # 버전 확인 (동일)
$ git config --list  # 설정 확인
$ git --help       # 도움말
$ pwd              # 현재 디렉토리
$ ls -al           # 파일 목록 (전체)
```
